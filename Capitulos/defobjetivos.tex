\chapter{Definición de objetivos}\label{cap:objetivos}

\section{Objetivo general}\label{sec:objetivo-general}

El objetivo general de este Trabajo de Fin de Grado es diseñar, implementar y evaluar un sistema software para analizar y predecir la volatilidad de datos financieros intradía. El trabajo no pretende construir una herramienta de inversión ni anticipar la dirección del precio, sino desarrollar un procedimiento reproducible que permita transformar datos de mercado en información útil sobre la intensidad esperada de los movimientos futuros.

El proyecto combina tres elementos. En primer lugar, un procedimiento de análisis que parte de datos brutos, los valida, construye variables de trabajo y compara distintos modelos predictivos. En segundo lugar, una comprobación con un caso sintético conocido, que permite observar el comportamiento del procedimiento en un entorno controlado antes de aplicarlo a datos reales. En tercer lugar, una aplicación web que utiliza una parte de los artefactos generados durante el estudio y ofrece una interfaz sencilla para cargar datos, ejecutar predicciones y visualizar resultados.

La aplicación empírica se centra en Bitcoin, utilizando datos de mercado con frecuencia de cinco minutos. A partir de ellos se estudia la volatilidad futura a corto plazo. Esta elección permite abordar un problema financiero real sin convertir el trabajo en una estrategia de trading. El interés principal está en la construcción del sistema, la comparación entre modelos y la interpretación prudente de los resultados obtenidos.

\section{Objetivos específicos}\label{sec:objetivos-especificos}

Para alcanzar el objetivo general se plantean los siguientes objetivos específicos:

\begin{enumerate}
     \item Establecer el marco conceptual necesario para interpretar el problema, diferenciando entre predicción de precio, predicción de volatilidad, análisis de patrones y límites de interpretación en datos financieros reales.
     \item Diseñar un procedimiento reproducible de análisis de datos financieros intradía, desde la preparación de los datos hasta la comparación de modelos predictivos y la generación de resultados documentables.
     \item Comprobar el comportamiento del procedimiento en un caso sintético conocido, de forma que exista un punto de referencia controlado antes de aplicarlo a datos financieros reales.
     \item Aplicar el procedimiento a Bitcoin, construyendo una medida de volatilidad futura a corto plazo y evaluando si los datos recientes contienen información útil para estimarla.
     \item Comparar modelos predictivos de distinta complejidad bajo una separación temporal adecuada, evitando que la evaluación dependa de una única técnica o de una referencia demasiado débil.
     \item Implementar un MVP web que integre una parte de los resultados del estudio y permita cargar datos, calcular variables, obtener predicciones de volatilidad y visualizar las salidas principales de forma comprensible. 
\end{enumerate}
\section{Alcance del trabajo}\label{sec:alcance}

El alcance del trabajo incluye el desarrollo de un procedimiento completo de análisis y predicción de volatilidad a corto plazo. Este procedimiento cubre la obtención de datos, su validación, la construcción de variables, la comparación de modelos, la generación de resultados y la exportación de artefactos reutilizables.

El estudio se aplica a dos contextos. El primero es un caso sintético conocido, usado como control del procedimiento. El segundo es una aplicación empírica sobre Bitcoin, utilizando datos intradía del par \texttt{BTCUSDT}. Otros activos, otras frecuencias y otros horizontes de predicción quedan fuera del alcance principal y se consideran extensiones futuras.

El trabajo incluye también el desarrollo de un MVP web. Este prototipo permite utilizar una parte del sistema en un entorno interactivo, pero no reproduce todo el análisis experimental. Su finalidad es demostrar que los modelos y artefactos generados pueden integrarse en una herramienta software utilizable.

Quedan fuera del alcance del proyecto la construcción de una estrategia de inversión, la ejecución de operaciones reales, la optimización de carteras, la gestión de riesgo financiero y la recomendación de compra o venta de activos. El sistema se limita a estimar volatilidad futura y a mostrar los resultados de forma interpretable.

\section{Limitaciones del trabajo}\label{sec:limitaciones-objetivos}

El trabajo presenta limitaciones que deben tenerse en cuenta desde el inicio. La primera es que los datos financieros reales son ruidosos y cambian con el tiempo. Por tanto, cualquier patrón observado debe interpretarse con cautela y compararse con modelos de referencia antes de extraer conclusiones.

La segunda limitación afecta a la predicción. El sistema estima volatilidad futura, no rentabilidad ni dirección del precio. Una predicción de mayor volatilidad no indica por sí sola si el precio subirá o bajará. En consecuencia, los resultados no deben entenderse como señales operativas de mercado.

La tercera limitación está relacionada con el alcance empírico. La aplicación principal se realiza sobre Bitcoin, con frecuencia de cinco minutos y horizonte de una hora. Los resultados obtenidos no pueden generalizarse automáticamente a otros activos, periodos o frecuencias sin repetir el análisis.

La cuarta limitación corresponde al MVP web. El prototipo demuestra la integración software de una parte del estudio, pero no incorpora todas las características de una plataforma profesional. No incluye ejecución automática de operaciones, gestión de usuarios, despliegue productivo, monitorización continua ni estimación formal de incertidumbre.

Por último, el trabajo distingue entre indicios y conclusiones. Que un modelo mejore a otro en una muestra concreta no implica que exista una explicación completa del comportamiento del mercado. La memoria debe mantener esta diferencia durante todo el análisis.

\section{Aportación principal del TFG}\label{sec:aportacion-principal}

La aportación principal del TFG consiste en integrar análisis de datos financieros, comparación predictiva e implementación software dentro de un mismo sistema reproducible. El trabajo no se limita a probar un modelo aislado, sino que diseña una cadena completa desde los datos brutos hasta una aplicación web funcional.

Desde el punto de vista metodológico, la aportación está en construir un procedimiento ordenado para estudiar la volatilidad futura a corto plazo. Este procedimiento incluye un caso sintético de control, una aplicación empírica sobre Bitcoin y una comparación entre modelos con distinta complejidad.

Desde el punto de vista de Ingeniería de Software, la aportación consiste en organizar el proyecto en componentes diferenciados: repositorio de análisis, artefactos exportables, MVP web, pruebas y documentación. Esta separación permite mantener trazabilidad entre los resultados generados y la herramienta que los utiliza.

En conjunto, el TFG aporta una solución que conecta investigación aplicada y desarrollo software. El resultado no es una plataforma financiera profesional, sino un sistema académico reproducible que permite analizar la volatilidad de Bitcoin, comparar modelos y mostrar una parte de los resultados mediante una interfaz web.
